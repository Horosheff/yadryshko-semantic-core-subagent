# Методология Core

Этот документ — короткая методология, которую Core использует при сборке семантического ядра.

## Что такое семантическое ядро

Семантическое ядро — это не просто список ключевых фраз. Это рабочая карта:

- какие запросы есть у аудитории;
- какой у них интент;
- какие запросы относятся к одной теме;
- какой URL должен закрывать каждый кластер;
- какие страницы нужно создать, обновить, объединить или не трогать;
- какие задачи получает редактор, SEO, разработчик и владелец бизнеса.

## Основной pipeline

1. Бриф и границы бизнеса.
2. Инвентаризация сайта.
3. Seed-семейства.
4. Wordstat-сбор.
5. Очистка запросов.
6. Интент.
7. Кластеризация.
8. URL-карта.
9. Контент-брифы.
10. GEO/AI-слой.
11. Roadmap.
12. Большой HTML-отчёт.
13. Excel-книга.

## Интенты

- `I` — информационный: человек хочет узнать.
- `C` — коммерческий обзор: человек сравнивает варианты.
- `T` — транзакционный: человек хочет купить, заказать, записаться.
- `N` — навигационный: человек ищет конкретный бренд/сайт.
- `Mixed` — смешанный интент.
- `Unknown` — нужна ручная проверка.

## Правило кластера

Один кластер = один доминирующий интент + один основной URL.

Если запросы похожи словами, но в выдаче требуют разных страниц, их нужно разделить. Например:

- “что такое X” — статья;
- “X курс” — коммерческая посадочная;
- “X цена” — услуга/продукт;
- “X отзывы” — коммерческий обзор или блок доверия.

## Приоритеты

- `P0` — делать первым: деньги, лиды, стратегия, большой спрос.
- `P1` — важные страницы ближайшего этапа.
- `P2` — backlog.
- `P3` — наблюдение или низкий приоритет.

## URL-статусы

- `ok` — URL уже подходит.
- `refresh` — URL есть, но нужно усилить.
- `create` — нужна новая страница.
- `merge` — объединить страницы/кластеры.
- `301` — редирект.
- `blocked` — не продвигать по бизнес/правовым причинам.

## GEO/AI-слой

Для информационных и смешанных кластеров Core добавляет:

- краткие определения;
- FAQ;
- вопросы для H2/H3;
- таблицы сравнения;
- блоки доверия;
- источники и обновляемость;
- структуру, удобную для AI-ответов и фрагментов выдачи.

## Что нельзя делать

- Нельзя выдумывать частотности.
- Нельзя скрывать, что SERP не проверялся.
- Нельзя смешивать разные интенты ради красивой таблицы.
- Нельзя отдавать только CSV без понятного HTML-отчёта.
- Нельзя заявлять, что работа production-ready, если нет SERP/GSC/Яндекс.Вебмастера.
